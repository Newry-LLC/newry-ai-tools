# Primary Research Toolkit — Coverage & Gap Analysis

**Status: Absorbed into Interview Coding & Synthesis (ICS) Mode 2 — 2026-05-04**

This sub-skill was originally planned as a separate analysis pass after coding. On review (2026-05-04), the ICS Mode 2 Roll-up was found to already produce most of what the original CGA scope described — coverage table with substantive/partial/not-addressed counts, an Overall ✓/~/— rating that reflects volume and source mix, per-branch Gaps subsection, contradictions, Outside the Issue Tree, and a Key limitation cell. Keeping CGA as a separate skill mainly created duplication risk.

The four pieces that the original CGA was going to add — but that ICS Mode 2 doesn't do today — are tracked as ICS Mode 2 enhancements (not a separate sub-skill):

1. **Cross-round delta** — Mode 2 takes a prior Roll-up as optional input and calls out which branches strengthened, weakened, or were newly addressed since the prior round.
2. **Source-type recommendations for thin branches** — e.g., "B1 has only one supplier voice; talk to a customer."
3. **Cross-branch coverage commentary** — synthesized observations like "the corpus skews internal-NA; EU and voice-of-customer branches are systematically thin." Beyond what the per-row Key limitation cell does today.
4. **Optional integrated mode** — accepts a secondary-research or prior-analysis summary as additional input for a more integrated coverage view. Lower priority — interview-only mode is highly load-bearing on its own.

Companion ICS enhancements that support incremental rounds:

- **Card freeze marker** — hand-edited summary cards are respected on re-runs.
- **Locked-scope persistence across rounds** — already implicit in ICS but should be made explicit so frame doesn't drift between rounds.

This folder is kept (rather than deleted) so the absorption decision and its design constraints remain discoverable. See `auto-memory/project_cga_design_guidance.md` for the underlying guidance.
