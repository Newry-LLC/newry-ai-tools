# Eval Run Log Format — SoF Toolkit

Use this structure when recording an eval run. Create a new file for each run (e.g., `SoF Skill Eval Run 2.md`, `SoF Skill Eval Run 3.md`).

## Per test case: two sections

### Quick Take
5–6 tight bullets distilling the essentials. Each bullet starts with a bolded label (e.g., **Headline:**, **Format:**, **Overall:**). Long enough to understand at a glance, short enough to skim.

### Full Eval
The detailed output. Four sections: **What's working** / **Top improvements** / **Headline rewrite** / **Consistency pass**. Lead with the point in each bullet — cut setup sentences. Follow with **Observations on Skill Performance** noting anything new or worth tracking across runs.
